# Plyr Agent Instructions

## Project Overview

Plyr is a simple, accessible, and customizable HTML5, YouTube, and Vimeo media player. This fork (`QuadDarv1ne/plyr-video`) extends support for additional providers (Rutube, VK, Mail.ru, Yandex Video, MTS Link) and includes transcription/translation features.

## Tech Stack

- **Language**: Vanilla ES6+ JavaScript (no frameworks)
- **Build**: Gulp + Rollup + Babel
- **Styles**: Sass (Dart Sass)
- **Tests**: Vitest with jsdom environment
- **Linting**: ESLint (@antfu/eslint-config), Stylelint
- **Package Manager**: pnpm (preferred) or npm

## Code Conventions

### JavaScript
- Use ES6 modules (`import`/`export`). Avoid CommonJS.
- Single quotes for strings.
- 2-space indentation.
- Semicolons required.
- No `if-newline` rule (keep compact conditionals).
- Globals available: `Hls`, `jQuery`, `Plyr` (browser environment).
- Keep functions small and focused. Avoid massive exported objects.

### Architecture
- `src/js/plyr.js` — Main `Plyr` class. Maintain backward compatibility for public API.
- `src/js/plugins/*.js` — Provider integrations (YouTube, Vimeo, Rutube, VK, etc.).
- `src/js/utils/*.js` — Utility modules (is, elements, events, strings, etc.).
- `src/js/config/defaults.js` — Default configuration.
- `src/js/controls.js` — UI controls logic (legacy, TODO: refactor into smaller files).

### SCSS
- Located in `src/sass/`.
- Uses `@import` for partials (not `@use` yet).
- Class naming is BEM-like: `plyr` block, `plyr__*` elements, `plyr--*` modifiers.
- Partials are organized by component: `components/controls`, `components/menus`, etc.

### Providers
- New providers should extend the embed pattern in `src/js/plugins/`.
- Each provider must expose URL parsing and API initialization logic.
- Add tests in `test/providers/<name>.test.js`.

## Build & Development

### Scripts
```bash
pnpm start          # Start development with BrowserSync
pnpm run build      # Build JS and CSS for production
pnpm run lint       # Run ESLint + Stylelint + remark
pnpm run lint:fix   # Auto-fix lint issues
pnpm test           # Run Vitest once
pnpm run test:watch # Run Vitest in watch mode
```

### Important Files
- `gulpfile.js` — Entry point, delegates to `tasks/build.js` and `tasks/deploy.js`.
- `build.json` — Build matrix for JS bundles (formats, polyfills, namespaces).
- `vitest.config.js` — Test config (globals enabled, jsdom environment).
- `eslint.config.mjs` — Flat ESLint config based on @antfu.

## Testing

- Tests live in `test/` and use Vitest.
- Provider tests verify URL parsing and ID extraction.
- Coverage includes `src/js/**/*.js` (V8 provider).
- Run tests before committing.

## Guidelines

- **Backward Compatibility**: Do not break the existing public `Plyr` API.
- **Modern Browsers Only**: Browserslist is `> 1%`.
- **Accessibility**: Maintain ARIA attributes and keyboard support.
- **Performance**: Avoid heavy DOM operations in hot paths.
- **Icons**: SVG sprite loaded from CDN or local path via `loadSprite`.
- **i18n**: Use `i18n` utility for user-facing strings.

## What to Avoid

- Do not introduce jQuery or other DOM libraries.
- Do not use TypeScript in source files (only `plyr.d.ts` for types).
- Do not modify `dist/` directly — it is generated by Gulp.
- Do not skip linting before committing.
