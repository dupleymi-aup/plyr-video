# Browsersync UI Documentation

Документация по Browsersync — инструменту для локальной разработки и мониторинга Plyr.

## Что такое Browsersync

Browsersync — это инструмент для синхронизации браузеров, автоматического обновления страниц (Live Reload) и удалённой отладки при разработке веб-приложений. В проекте Plyr он используется для:

- **Автоматической сборки** — пересборка JS/CSS/SVG при изменении исходных файлов
- **Live Reload** — мгновенное обновление страницы в браузере при изменении файлов
- **Тестирования на нескольких устройствах** — доступ к dev-серверу по локальной сети
- **Отладки** — встроенная UI-панель для мониторинга и управления

## Browsersync UI Панель

Откройте `http://localhost:3008` в браузере для доступа к панели управления.

### Overview

Главная страница панели показывает:
- **Статус сервера** — работает ли dev-сервер
- **Подключённые устройства** — список браузеров/устройств, подключённых к серверу
- **Активные соединения** — количество активных WebSocket соединений
- **Сетевой трафик** — статистика запросов

### Remote Debug

Панель для удалённой отладки подключённых устройств:
- **Список устройств** — все открытые вкладки с демо-страницей
- **Скриншоты** — превью состояния каждого устройства
- **Управление** — возможность перезагрузить все устройства одновременно

### Network Throttling

Эмуляция различных скоростей сети для тестирования производительности:

| Режим | Описание |
|-------|----------|
| `Offline` | Полное отключение сети |
| `GPRS` | ~50 Kbit/s — очень медленное соединение |
| `Regular 2G` | ~250 Kbit/s |
| `Good 2G` | ~450 Kbit/s |
| `Regular 3G` | ~750 Kbit/s |
| `Good 3G` | ~1.5 Mbit/s |
| `DSL` | ~2 Mbit/s |
| `Wi-Fi` | ~30 Mbit/s |

Полезно для тестирования:
- Скорости загрузки видео
- Работы превью-миниатюр при медленном соединении
- Поведения плеера при буферизации

### History

История навигации в dev-сервере:
- URL, по которым переходили подключённые устройства
- Временные метки посещений
- Возможность быстрого перехода к любому URL

### Info

Техническая информация о подключении:
- **Server** — версия Browsersync
- **Protocol** — используемый протокол (ws, wss)
- **Latency** — задержка соединения
- **User Agent** — информация о браузере/устройстве

### Plugins

Управление плагинами Browsersync:

| Плагин | Описание | Статус в Plyr |
|--------|----------|---------------|
| `Scroll` | Синхронизация прокрутки между устройствами | Отключён (`ghostMode: false`) |
| `Clicks` | Синхронизация кликов между устройствами | Отключён |
| `Forms` | Синхронизация действий в формах | Отключён |
| `Forms (inputs)` | Синхронизация ввода в полях | Отключён |
| `Forms (toggles)` | Синхронизация переключателей | Отключён |
| `Forms (submit)` | Синхронизация отправки форм | Отключён |

Ghost Mode отключён в конфигурации Plyr, чтобы избежать нежелательной синхронизации при тестировании.

## Функции разработки

### Live Reload

Автоматическое обновление страницы при изменении файлов:

| Тип файлов | Действие при изменении |
|------------|----------------------|
| `src/js/**/*.js` | Пересборка JS → обновление страницы |
| `src/sass/**/*.scss` | Пересборка CSS → обновление страницы |
| `src/sprite/*.svg` | Пересборка SVG спрайта → обновление страницы |
| `demo/src/js/**/*.js` | Пересборка демо JS → обновление |
| `demo/src/sass/**/*.scss` | Пересборка демо CSS → обновление |

**Процесс:**
1. Вы сохраняете файл
2. Gulp обнаруживает изменение (watch)
3. Запускается соответствующая задача сборки
4. Browsersync получает уведомление о новом файле
5. Страница обновляется автоматически (inject CSS без полной перезагрузки)

### Ghost Mode

Синхронизация действий между несколькими подключёнными устройствами/вкладками:

- **Scroll** — прокрутка на одном устройстве повторяется на всех
- **Clicks** — клики синхронизируются
- **Forms** — ввод в формах синхронизируется

В Plyr отключён (`ghostMode: false`), так как при тестировании видеоплеера синхронизация не требуется.

### Watch (Наблюдение)

Gulp отслеживает изменения файлов в реальном времени:

```js
// Из tasks/build.js
gulp.watch(paths.plyr.src.js, js);         // src/js/**/*.js
gulp.watch(paths.plyr.src.sass, css);      // src/sass/**/*.scss
gulp.watch(paths.plyr.src.sprite, sprites); // src/sprite/*.svg
gulp.watch(paths.demo.src.js, js);         // demo/src/js/**/*.js
gulp.watch(paths.demo.src.sass, css);      // demo/src/sass/**/*.scss
```

## Конфигурация в gulpfile.js

Browsersync настраивается в функции `serve()` в `tasks/build.js`:

```js
import browserSync from 'browser-sync';

const bs = browserSync.create();

export function serve() {
  return bs.init({
    server: {
      baseDir: paths.demo.root,  // demo/ — корень сервера
    },
    notify: false,    // Не показывать уведомления "Connected to Browsersync"
    watch: true,      // Следить за изменениями файлов
    ghostMode: false, // Отключить синхронизацию между устройствами
  });
}
```

### Параметры конфигурации

| Параметр | Значение | Описание |
|----------|----------|----------|
| `server.baseDir` | `demo/` | Директория, которую обслуживает сервер |
| `notify` | `false` | Отключить всплывающие уведомления |
| `watch` | `true` | Автоматически обновлять при изменении файлов |
| `ghostMode` | `false` | Отключить синхронизацию действий |

### Пути проекта

```js
const paths = {
  plyr: {
    src: {
      sass: 'src/sass/**/*.scss',
      js: 'src/js/**/*.js',
      sprite: 'src/sprite/*.svg',
    },
    output: 'dist/',
  },
  demo: {
    src: {
      sass: 'demo/src/sass/**/*.scss',
      js: 'demo/src/js/**/*.js',
    },
    output: 'demo/dist/',
    root: 'demo/',
  },
};
```

## Gulp Tasks

### Основные задачи

| Задача | Описание |
|--------|----------|
| `gulp` (default) | Полная сборка + запуск сервера + watch |
| `gulp build` | Полная сборка (clean + js + css + sprites) |
| `gulp clean` | Очистка директорий `dist/` и `demo/dist/` |
| `gulp js` | Сборка всех JS файлов |
| `gulp css` | Сборка всех CSS файлов |
| `gulp sprites` | Сборка SVG спрайтов |
| `gulp serve` | Запуск Browsersync сервера |
| `gulp watch` | Запуск наблюдения за файлами |

### Процесс сборки

```
default → build → clean
                → js (parallel)
                   → js:plyr.js:es
                   → js:plyr.js:umd
                   → js:plyr.polyfilled.js:es
                   → js:plyr.polyfilled.js:umd
                   → js:demo.js:iife
                → css (parallel)
                   → css:plyr.css
                   → css:demo.css
                   → css:error.css
                → sprites (parallel)
                   → sprite:plyr.svg
                   → sprite:demo.svg
        → serve + watch (parallel)
```

### Смена порта

По умолчанию Browsersync использует порт 3007. Для изменения отредактируйте `tasks/build.js`:

```js
export function serve() {
  return bs.init({
    server: { baseDir: paths.demo.root },
    port: 8080,        // Изменить порт сервера
    ui: { port: 8081 } // Изменить порт UI
    notify: false,
    watch: true,
    ghostMode: false,
  });
}
```

## Мониторинг

### Что мониторить

| Показатель | Где смотреть | Описание |
|------------|-------------|----------|
| Сборка JS | Консоль gulp | Размеры файлов, ошибки компиляции |
| Сборка CSS | Консоль gulp | Размеры файлов, warnings Sass |
| Спрайты | Консоль gulp | Количество оптимизированных изображений |
| Подключённые устройства | Browsersync UI → Overview | Количество активных соединений |
| Сетевая задержка | Browsersync UI → Info | Latency в мс |
| История запросов | Browsersync UI → History | Запрошенные ресурсы |

### Типичные warnings

| Warning | Значение | Действие |
|---------|----------|----------|
| `Sass @import rules are deprecated` | Sass v3 готовит удаление @import | Миграция на @use/@forward (не критично) |
| `lighten() is deprecated` | Функция цвета устарела | Использовать `color.adjust()` (не критично) |
| `Circular dependency` | Циклический импорт в JS | captions.js ↔ controls.js (не критично) |
| `AWS SDK v2 end-of-support` | Старая версия AWS SDK | Миграция на v3 (не критично) |
